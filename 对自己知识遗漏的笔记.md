#  Video

## 1. 基础语法 以及 核心属性
```html
<video autoplay muted loop>
    <source src="video.webm" type="video/webm"> 
    <source src="video.mp4" type="video/mp4">   
</video>

```

### 核心属性表

| 属性 | 功能描述 | 实战建议 |
| :--- | :--- | :--- |
| `autoplay` | 自动播放 | 背景视频必备属性。 |
| `muted` | **静音** | **关键：** 现代浏览器要求自动播放必须静音，否则会被拦截。 |
| `loop` | 循环播放 | 适合氛围背景视频，让画面无缝衔接。 |
| `playsinline` | 行内播放 | 防止在 iOS 端自动弹出全屏播放，保持页面布局完整。 |
| `poster` | 封面占位图 | 视频加载完成前显示的图片，防止因网络慢导致页面留白。 |
| `controls` | 显示控制条 | 纯背景视频通常**不加**此属性，保持视觉干净。 |
| `object-fit` | 视频填充方式 | CSS 属性，常用 `cover` 实现像背景图一样的铺满效果。 |
## 2. 基础结构
使用 `<source>` 标签可以提供多种格式，增强浏览器的兼容性。



```html
<video width="100%" height="auto" poster="images/cover.jpg">
    <source src="video/brand-story.mp4" type="video/mp4">
    <source src="video/brand-story.webm" type="video/webm">
    您的浏览器不支持 HTML5 视频播放。
</video>

```

# tips:使用`<source>`的原因

在 HTML5 中，虽然可以直接在 `<video>` 标签上写 `src` 属性，但职业开发者更倾向于使用内部的 `<source>` 标签。其核心原因可以概括为以下

---

### 解决浏览器兼容性 (Browser Compatibility)
不同的浏览器对视频格式的支持情况不同。如果没有 `<source>`，你的视频可能会在某些设备上无法播放。

* **MP4 (H.264)**：兼容性之王，几乎所有浏览器都支持。
* **WebM**：Google 推出的格式，画质高体积小，但在旧版 Safari 上支持欠佳。

**作用机制：** 浏览器会从上到下扫描 `<source>`，直到找到一个它能识别的格式并立即播放，忽略后面的内容。

```html
<video autoplay muted loop>
    <source src="video/haitian.webm" type="video/webm">
    <source src="video/haitian.mp4" type="video/mp4">
</video>

```

### 🕶️ CSS 3D 透视笔记
- **属性**：`perspective`
- **原理**：模拟人眼观察物体的“近大远小”规则。
- **参数值**：常用 `800px` 到 `1200px`。
- **核心逻辑**：
    1. 父元素设 `perspective`（开透视）。
    2. 子元素设 `transform: rotate`（做旋转）。
    3. 子元素设 `transform-style: preserve-3d`（保 3D 环境）。

## 案例

### 案例一：丝滑的产品卡片翻转 (Card Flip)
这是品牌官网展示产品（如酱油瓶背面配料表）最常用的交互。

**HTML:**
```html
<div class="flip-container">
    <div class="flip-card">
        <div class="front"><img src="sauce-front.jpg" alt="正面"></div>
        <div class="back">有机酿造，零添加</div>
    </div>
</div>
```


**css:**
```css
.flip-container {
    perspective: 1000px; /* 1. 开启透视眼 父元素专用 */
    ...
}

.flip-card {
    position: relative;
    transform-style: preserve-3d; /* 2. 保持内部 3D 环境 子元素专用*/
}

/* 鼠标悬浮翻转 180 度 */
.flip-container:hover .flip-card {
    transform: rotateY(180deg);
}

.front, .back {
    position: absolute;
    width: 100%;
    height: 100%;
    backface-visibility: hidden; /* 3. 翻到背面时隐藏正面 */
}

.back {
    background-color: #f4f4f4;
    transform: rotateY(180deg); /* 初始状态背面就是反的 */
}
```
**小tips: 当想提高动画的性能时 可以通过translate3d(0,0,0) 实现加速**

# 📘 鼠标绑定回到顶部的功能：`scrollTo` 与 `behavior` 详解

### 1. `window.scrollTo()` 方法定义
`scrollTo()` 是 Web API 提供的接口，用于将浏览器窗口或特定容器滚动到指定的坐标位置。

---

### 2. 语法格式

#### **A. 坐标模式 (立即跳转)**
这是最基础的写法，适用于不需要动画的场景。
```javascript
// 语法：window.scrollTo(x-coord, y-coord)
window.scrollTo(0, 0); // 瞬间回到页面左上角
```
### 3. 具体实现的思路
```javascript
//首先在h5里面绑定一个图标 例如目前他的id为backtotop 则在js里面实现为
DocumentTimeline.getElementById('backToTop').addEventListener('click',function(){
    window.scrollTo{
        top:0;
        behavior:'smooth',
    }
});
//注意这里的behavior存在2个值 1.auto 表示默认的滑动 2.smooth比较丝滑的移动
```


# CSS 布局

## 1.  Grid 布局（网格布局）

**Grid 布局**是 CSS 中最强大的 **二维布局系统**。它改变了我们设计用户界面的方式，允许我们将页面划分为行和列。

* **核心理念**：将容器划分为“行”和“列”组成的网格，通过定义网格轨道（Tracks）来摆放内容。
* **适用场景**：复杂的全页结构设计、规则或不规则的卡片阵列（如 2x2 或 3x3 布局）。

### Grid 的关键特性：
* **二维控制**：同时管理水平方向（行）和垂直方向（列）。
* **直观的行列定义**：使用 `grid-template-columns` 和 `grid-template-rows` 快速定型。
* **内置间距控制**：通过 `gap` 属性直接设置格子间的距离，无需计算 `margin`。
* **重叠能力**：允许不同元素占据同一个网格单元格，实现复杂的叠加视觉。



---

## 2. 常见布局方式对比

除了 Grid，前端开发中还有以下几种主流布局手段：

### ① Flexbox 布局（弹性盒子）
* **维度**：**一维布局**（一次只能处理行或列中的一种）。
* **适用场景**：导航栏文字排布、轮播图横向排列、垂直居中某个元素。
* **特点**：强于内容在轴线上的对齐（Align）和分布（Justify）。

### ② Position 定位（绝对/相对定位）
* **维度**：坐标定位。
* **适用场景**：悬浮图标、装饰红线（如 `::after` 伪元素）、固定在顶部的菜单。
* **特点**：元素脱离标准文档流，不干扰其他元素的位置。

### ③ Float 布局（浮动）
* **维度**：一维（水平）。
* **适用场景**：旧版网页布局、文字环绕图片。
* **特点**：目前已基本退出主流整体布局舞台，主要用于简单的图文混排。

### ④ Table 布局（表格）
* **适用场景**：仅用于展示财务报表等真正的结构化数据。
* **特点**：代码冗长且死板，不利于移动端响应式适配。

---

## 3. 布局选择指南：我该用哪个？

在现代网页开发中，推荐遵循 **“大架子用 Grid，小细节用 Flex”** 的原则：

| 布局方式 | 维度特性 | 典型应用建议 |
| :--- | :--- | :--- |
| **Grid** | 二维 (行+列) | **整体框架**：如页面大板块划分、多行多列的卡片区。 |
| **Flex** | 一维 (横或纵) | **局部组件**：如导航菜单、一行内的图标文字对齐。 |
| **Position** | 坐标点定位 | **特殊装饰**：如角标、提示框、下划线特效。 |

> **一句话总结**：如果你要画的是“棋盘”，用 **Grid**；如果你要排的是“队伍”，用 **Flex**。



# Grid 语法

## 1. Grid 核心概念图解

Grid 布局将页面划分为“容器”和“项目”。通过在容器上定义网格轨道，在项目上控制跨度。



---

## 2. 常用语法速查

### ① 容器属性（作用在父元素）
用于定义网格的结构：

| 属性 | 说明 | 示例 |
| :--- | :--- | :--- |
| `display: grid;` | 激活网格布局 | `display: grid;` |
| `grid-template-columns` | 定义列的数量和宽度 | `repeat(2, 1fr)` (平分两列) |
| `grid-template-rows` | 定义行的数量和高度 | `auto auto` (根据内容自适应) |
| `gap` | 设置格子之间的间距 | `gap: 20px;` |

### ② 项目属性（作用在子元素）
用于控制单个方块的摆放位置：

* **`grid-column`**: 定义项目从哪条线开始，到哪条线结束。例如 `1 / 3` 表示占用前两个格子。
* **`grid-row`**: 定义垂直方向的跨度。

---

## 3. 实战案例：海天厨师活动 2x2 布局

针对 **厨师活动区域** 的四宫格排版，最简洁的 Grid 实现方式如下：

### HTML 结构
```html
<div class="grid-container">
    <div class="item">参与厨师活动</div>
    <div class="item">免费领样</div>
    <div class="item">共创星厨菜谱</div>
    <div class="item">加入厨师社区</div>
</div>
```

### css 结构
```css
.grid-container {
    display: grid;
    /* 核心：创建两列，每列平分 1 份剩余空间 */
    grid-template-columns: repeat(2, 1fr); 
    /* 定义两行，高度根据内容自动撑开 */
    grid-template-rows: auto auto; 
    /* 设置格子间的距离 */
    gap: 30px; 
    
    max-width: 1200px;
    margin: 0 auto;
}

.item {
    background: #fff;
    border-radius: 12px;
    padding: 20px;
    box-shadow: 0 4px 10px rgba(0,0,0,0.1);
}
